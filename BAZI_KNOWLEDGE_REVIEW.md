# Bazi Knowledge Engineering v0.1 Review

## 范围与来源

首批仅覆盖日主、十神、五行旺衰、格局、用神、喜忌、大运和流年的基础解释边界。来源限于只读的 `spec/rules/base/` 三个 JSONL 文件；注册表以文件 SHA-256 生成稳定 source_id，并以行号作为原始位置。没有导入 PDF，也没有补造排盘、古籍引文或页码。

## Tianfu Agent 四层结构适配

本批支持四层知识结构：

1. Concept：8 个术语卡，定义分析对象和使用边界。
2. Rule：5 条候选规则，约束强弱、十神、财官印食伤、事件推断和现实优先级。
3. Evidence：5 条 exclusion records，明确禁止发财、婚姻、疾病及现实覆盖等错误提升。
4. Benchmark：24 个 draft 边界案例，将上述禁止项转换成可重复验收的输入与预期。

四层均带 source_id、原始行位置和生命周期；Benchmark 通过 `exclusion_id` 回链 Evidence。该结构支持 Tianfu Agent 的“概念—规则—证据—评测”知识闭环，但当前只完成边界层，不构成完整八字推演体系。

## Review 决策

可以进入 `reviewed` 的内容：

- 术语作为工作标签及其“不直接等于现实事件”的边界。
- 日主强弱需要分数、同异党比较和作用关系的输入要求。
- 十神只表达主题或倾向，必须结合承载、组合、制化与现实资料。
- 旺衰不能单独预测事件。
- 已确认现实事实和硬条件优先。

只能保持 `source_only` 的内容：

- 日主强弱阈值与具体判定法，目前只由仓库内规则支持。
- 财、官杀、印、食伤的细项解释与组合解释。
- 格局、用神、喜忌的选择或判定方法。
- 大运、流年的作用方式与事件窗口。
- 任何发财、婚姻、疾病、灾祸或具体年份结论。

现实证据优先规则有多条现有 reality rules 支持，标记为 `corroborated`；为避免本批直接影响生产行为，仍设置 `production_allowed=false`。其余规则均为 `source_only`。所有对象保持 reviewed 或 draft，绝不自动升级 verified。

## 已知限制

- `spec/rules/base/` 是产品内部规范，不是独立学术或古籍证据。
- 当前来源不足以验证格局、用神、喜忌、大运和流年的完整方法论，因此仅保留概念和安全边界。
- 本批不评价命理预测准确率，也不提供自动排盘或个人事件判断。
