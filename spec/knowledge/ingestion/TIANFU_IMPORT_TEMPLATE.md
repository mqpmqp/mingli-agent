
# Tianfu 资料导入模板

当前包尚未取得 GitHub 中的天府蒸馏资料正文，因此只提供导入合同，不声称已经吸收。

## 最少需要

- 仓库全名或导出的文件包
- 文件路径与版本/commit
- 方法说明
- 规则或案例来源
- 许可与使用边界

## 三类拆分

### Method
记录判断顺序、任务路由、证据组合和输出结构。

### Rule Candidate
转换为有 trigger / support / exclude 的候选规则，默认状态 `draft`。

### Case
记录盘面、现实背景、事件时间线与验证依据。

## 审核门槛

1. 不因来源名为 Tianfu 就自动提高置信度。
2. 每条规则必须定位到文件和行/段。
3. 与现有规则冲突时建立 conflict record。
4. 通过黄金案例或历史回测后才可升级为 verified。
