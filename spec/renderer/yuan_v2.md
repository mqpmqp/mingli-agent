# Yuan Renderer V2

## 基本要求

- 先给结论，再给依据。
- 结论按高、中、低置信分组。
- 术语后必须立刻给白话解释。
- 不展示规则编号、内部评分或推理草稿。
- 现实限制必须写入同一段结论，而不是放在末尾弱化。
- 结尾固定：仅供文化研究与娱乐参考。

## 完整命盘默认结构

### 1. 资料确认
列出出生资料、四柱、日主、当前大运和信息限制。

### 2. 称骨歌诀
仅在可靠算法已输出骨重时使用。不得自行心算或猜测。

### 3. 结论
分别列高置信、中置信、低置信判断。

### 4. 事业
写清能力结构、适合环境、现实限制与阶段变化。

### 5. 财运
写清财富路径，不写保证收益。

### 6. 感情
写关系模式、现实状态与稳定条件。

### 7. 五年断事
当前年 ±2 年，每年一句主断。
使用“较可能”“重点主题”“需防”等表达。
不写具体灾祸、死亡或保证性事件。

### 8. 建议
只给可执行、现实、低成本且可逆的建议。

## 专项结构

### 考公考编
1. 体制适配度
2. 能否上岸
3. 岗位方向
4. 备考策略

### 感情复合
1. 缘分牵引
2. 复联可能
3. 复合可能
4. 稳定可能

### 决策比较
至少比较两个方案：
- 优势
- 风险
- 现实前提
- 最小可逆行动
