# Metis 公开产品能力差距审查

日期：2026-07-16  
范围：仅观察 `https://metisziwei.com/chart` 公开可见的输入字段与用户流程。

## 合规边界与方法

本审查未爬取网站、未探测或逆向 API、未绕过登录、未复制输出文本/源码/知识库，也未把外部输出作为算法 Oracle 或训练数据。页面输出因交互超时未能可靠观察，相关项统一标为“无法验证”，不作推断。

## 仓库差距结论

基线搜索未发现紫微排盘、十二宫、星曜安置、四化、庙旺、规则、Runtime、Renderer 或 benchmark 实现；`knowledge` 中紫微目录仅有 `.gitkeep`。可安全复用的是历法/太阳时、结构化证据融合、Yuan 八段渲染和通用合同工具。

因此本任务不实现无可靠来源的传统排盘规则，而完成可验证的 P0 工程底座，并把星曜安置与传统规则保持显式 NO-GO。该结论优先于覆盖率数字。

## 风险

- 把通用时间修正等同于紫微排盘会产生伪确定性。
- 未知时辰若默认子时但不标降级，会制造错误命盘身份。
- 仅用用户 ID 或姓名做缓存键会导致串盘或身份字段变更后索引漂移。
- 异步结果没有 revision/context 校验会让旧请求覆盖新命盘。
- 未经来源审查录入传统解释会破坏 provenance 与发布门禁。
