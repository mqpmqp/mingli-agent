# 紫微出生数据隐私与治理

## 数据分类

出生日期、精确出生时间、出生地点、经纬度、性别、命盘 fingerprint 与案例时间线均按敏感个人资料处理。姓名、联系方式、账号标识不应进入命盘计算载荷或外部模型提示。

## 最小化与隔离

- 外部模型仅接收完成任务所需的结构化字段；默认移除姓名、精确地址、账号和原始对话。
- cache key 必须包含 opaque user scope、case scope、chart fingerprint、完整 temporal context 和 namespace。
- 日志仅记录 revision、状态码与截断 digest，不记录出生资料、现实事件正文或渲染全文。
- 导出前移除内部 user/case scope，并对时间线做再识别风险复核。

## 保存与缓存

- 未登录/未同意存储：仅会话内存，任务结束删除。
- 已同意存储：按明确 fixed-term policy；不得无限期默认保存。
- 结果缓存随 chart revision 切换失效；旧异步结果由 revision/context gate 拒绝。
- 普通咨询记录不自动成为训练或模型改进数据。

## 授权

存储、模型改进、公开案例三个 consent 字段相互独立，不能互相推定。匿名化状态不是公开授权。真实案例 benchmark 至少需要存储授权、匿名化完成且未撤回；模型改进还需单独明确授权。

## 删除与撤回

收到撤回后先将 `withdrawal_status` 标为 requested，停止读取和统计，完成物理/逻辑删除后标为 deleted，并清除相关缓存与导出物。不得继续把已删除案例计入 benchmark。

## 供应商与高风险输出

向模型供应商发送前执行字段白名单与敏感值脱敏。不得发送 Cookie、Token、密钥或内部 scope。健康、法律、投资和婚姻相关内容不能仅凭命理结果作决策，应提示现实核验与持证专业人士。
