# AGENTS.md — MingLi Agent

## 默认语言
- 文档、报告与 CLI 用户文本默认使用中文。
- 代码、类型名和模块名使用英文。

## 工作方式
- 先检查仓库、分支、工作树、Python 版本和现有测试。
- 使用最小、可验证的变更。
- 不做无关重构。
- 测试失败时先读错误、修复、重跑，直到通过或触发停止条件。
- 不声称执行过未实际运行的测试。

## 安全边界
- 不读取 `.env`，不打印环境变量、密钥或令牌。
- 不接入支付、真实用户数据、医疗、投资或生产服务。
- 不调用真实 OpenAI、GitHub、排盘或外部网络 API。
- 不实现未经验证的农历、节气、四柱、大运或紫微算法并声称准确。
- 不执行全书 OCR。
- 不自动升级 `draft` / `reviewed` 规则为 `verified`。
- 不修改 `spec/` 中的来源资产；如发现问题，写入报告。
- 不生成“必然、一定、注定、百分百、必死、必离婚、必发财”等保证性输出。

## 工程边界
- Python >= 3.11。
- 首版使用单体 Python package，不拆微服务。
- 首版只实现确定性的规范校验、规则加载、证据融合、现实校正、置信度和文本渲染。
- 排盘使用 `ChartProvider` 协议；没有可靠实现时必须显式拒绝精确分析。
- 图片盘未确认时只能返回低置信限制说明。
- 复合问题必须覆盖：缘分牵引、复联、复合、稳定。
- 考公问题必须覆盖：体制适配、上岸、岗位、备考。
- 免责声明固定只在答案末行出现一次。

## 交付要求
- 创建独立分支。
- 完成代码、测试、README 和实现报告。
- 运行全部验收命令。
- 提交并推送当前分支。
- 最终输出：分支、commit、改动文件、验证命令与结果、未实现边界和剩余风险。
