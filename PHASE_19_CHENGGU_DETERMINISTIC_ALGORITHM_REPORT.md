# Phase 19：称骨确定性算法

P19 已将一套常见称骨权重冻结为 `chenggu-common-table-r1@0.1`。计算输入统一落到农历年、月、日与本地民用时辰，四项均以整数“钱”求和，再格式化为“两/钱”，避免浮点误差。

## 固定口径

- 阳历输入使用项目既有 1900—2099 农历表做确定性逆转换。
- 年份按农历年干支取重，不使用八字立春年柱。
- 子时为 23:00—00:59，不区分早晚子时，不做真太阳时修正。
- 权重表只有 12 个月；闰月沿用同数字月份权重，并输出 `leap_month_uses_same_numeric_month_weight`。
- 输出固定为 `prediction_validity=not_evaluated`，不得解释为科学预测或确定事件。

## 数据来源与边界

权重表据现代二级资料 [八字重量表](https://www.ba-zi.ai/blog/bazi-weight-chart) 冻结。该资料也提示民间存在异表，因此本实现只承诺具名版本内的复现性，不宣称唯一权威。RC2 核心包按设计只提供算法与骨重，不包含歌诀原文、歌诀 package-data 或现代白话解释，故固定 `verse_available=false`。未来 optional verse pack 必须独立版本化和评审，不影响本算法状态。

## 验证

- 基准：甲子年正月初一子时为 3 两 9 钱。
- 阳历样例：1990-03-15 10:30 转为庚午年二月十九巳时，总重 3 两 7 钱。
- 覆盖表数量、阳历转农历、闰月约定、整数求和、输入拒绝与 canonical hash 稳定性。
